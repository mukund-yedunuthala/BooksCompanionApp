# Security Policy

## Reporting a Vulnerability

Use this section to tell people how to report a vulnerability.

Tell them where to go, how often they can expect to get an update on a
reported vulnerability, what to expect if the vulnerability is accepted or
declined, etc.

If you discover any security vulnerabilities, weaknesses, or other concerns within this repository, please report them to the project maintainer immediately. To report a security issue, follow these steps:

1. Create a new GitHub issue with the label "Security Issue."
2. Provide a clear and detailed description of the issue. Include steps to reproduce, if applicable.
3. The maintainer will acknowledge the issue and begin working on a solution.
4. Please allow a reasonable amount of time for the issue to be addressed before disclosing it publicly.
